# Protocol — Stage 1 & 2

## What & Why

Build the first real version of Protocol: a premium, local-first PWA for people managing research compound protocols.

The goal is not to build the full operating system at once. The goal is to ship the smallest product that is:

* beautiful
* trustworthy
* sticky
* obviously better than spreadsheets, notes, and clunky calculators

Stages 1 and 2 should produce a fast, installable app with three core jobs:

1. calculate dose precisely
2. log it instantly
3. tell the user what is active, due next, and running low

---

## Lean Product Principle

Do not overbuild.

The first shipped version should feel like:

* **the best-looking reconstitution calculator on the internet**
* plus **a simple protocol tracker**
* plus **light privacy**

Not:

* a complete biohacker platform
* a multi-page monster
* a backend-heavy architecture exercise

## Key Product Decisions

### 1. No backend in Stages 1–2

Everything is local-first.

That means:

* no API work
* no OpenAPI/codegen
* no session layer
* no auth

This is deliberate. It keeps build speed high.

### 2. No full “Cycle Canvas” yet

Replace the fancy timeline with a lean protocol summary:

* Active now
* Next due
* Clears by
* Running low in X days

This solves the real problem faster.

### 3. Encryption stays simple

Use one encrypted local payload or a very small number of payloads.

Do not build row-level encrypted architecture yet.

### 4. One-screen-first layout

Prefer a dense, premium single-screen shell with sections:

* Calculator
* Syringe / outputs
* Log
* Protocol summary

Tabs are fine. Complex routing is not necessary.

### What to Build

#### Client-Side Encryption

Create `src/lib/crypto.ts` with:

* AES-GCM encryption/decryption
* passphrase-derived key or local device key
* simple unlock flow on app open
* encrypt/decrypt logs + settings

Keep it simple:

* one master payload or a few grouped payloads
* re-encrypt on mutation

#### Zustand Store

Create `src/store/protocolStore.ts` with slices for:

* calculator
* logs
* protocol
* app settings / lock state

Avoid a giant undifferentiated store.

#### Active Protocols

Add a lean protocol model:

* compound
* dose
* unit
* frequency
* start date
* optional end date
* vial amount
* water amount

This powers:

* next due
* washout
* inventory

#### Next-Dose Countdown

Support simple templates:

* daily
* twice daily
* every other day
* weekly
* custom hours

Show:

* due in X
* overdue flag
* last logged

#### Washout Logic

Based on half-life:

* estimate time to near-clearance
* show countdown/date
* simple visual progress bar

No complicated pharmacokinetic modelling yet.

#### Inventory Countdown

Calculate:

* units per dose
* doses remaining
* days remaining
* estimated depletion date

This is high-value and should be in Stage 2.

#### Saved Protocol Templates

Allow:

* save current protocol
* reload later
* duplicate
* delete

Small feature, high utility.

#### Export

Add local export:

* CSV
* JSON

This should exist before paid plans, even if basic.
